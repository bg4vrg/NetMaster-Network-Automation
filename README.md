# NetMaster 网络自动化运维平台

当前版本：v2.54.5

NetMaster 是面向专网、内网、物理隔离环境的交换机自动化运维工具。项目以本地 Flask + SQLite + Netmiko 为核心，不依赖外部 CDN、远程 API 或互联网服务，适合在无法访问互联网的生产网环境中部署。

## 当前主要能力

- 设备资产管理：支持 H3C、Huawei、Ruijie 标识，支持新增、批量导入、排序、编辑、删除。
- 设备角色：只保留“接入交换机”和“备份设备”。接入交换机参与终端更新和备份，备份设备只参与配置备份。
- ACL 策略组管理：支持查询指定 ACL 组或交换机全部 ACL 策略组，默认维护 MAC ACL 4000。
- 端口绑定：支持 IP/MAC 绑定、解绑、端口描述查询/保存/清空、关键端口保护；快捷选择交换机后自动加载端口列表。
- SNMP 轻量状态：支持配置 H3C 只读团体名，端口查询时优先读取链路、速率、描述和错误包状态，配置级确认仍走 SSH。
- Excel 批量部署：按交换机和端口聚合同组记录，减少重复 SSH 登录。
- 终端更新：后台扫描接入交换机，把交换机当前绑定信息写入本地 `mac_bindings` 表。
- 终端列表：支持交换机名称、交换机 IP、端口、VLAN、模式、更新时间展示，支持搜索、冲突筛选和 CSV 导出。
- 终端漫游：支持跨交换机迁移，包含定位、旧端口解绑复核、新端口差异化部署、IP 冲突保护。
- 配置差异比对：基于本地备份文件查看两次配置的新增和删除行。
- 配置差异分类：差异结果按接口、VLAN、终端绑定/准入、ACL、路由和管理服务等类型聚合提示。
- 任务中心：集中查看自动备份、终端更新、批量部署、终端漫游等任务历史。
- 端口画像：按交换机端口聚合已绑定终端数量、VLAN、模式、最近确认时间、SNMP 实时状态和维护建议。
- 运行健康检查：检查资产、备份、终端绑定覆盖率、绑定新鲜度和关键系统参数。
- 终端漫游试运行：只生成迁移计划，不登录交换机、不下发配置。
- 自动备份：每日定时备份配置，支持备份成功后自动保存配置，并可在网页版本信息中关闭该行为。
- 凭据安全：交换机资产密码本地加密存储，旧明文记录启动时自动迁移。
- 数据备份与恢复：网页导出/导入 `net_assets.db`、`net_assets.key`、交换机资产 CSV 和已绑定终端 CSV。
- 离线绑定导入：从历史备份配置解析绑定记录，辅助补齐 `mac_bindings`。
- 深度在线健康检查：手动低并发 SSH 检查设备连通性。
- 日志告警采集：采集交换机 `display logbuffer`，支持风险评分、风险等级、分类统计、高频端口提取、处置建议、每日定时采集和网页查看最近分析。
- 告警中心：按每台交换机最新日志分析聚合，提供全网风险概览、Top 10 优先处理设备、筛选、折叠原始日志和排查命令建议。
- 告警闭环：支持告警确认、忽略、重新打开、备注和近 7 天趋势统计。
- 准入合规分析：支持上传 IP 地址管理平台、9200 平台和人工审批台账导出表，按“保护状态=保护”或“核心版本包含 9200”判定 9200 合法终端，默认全量分析导出文件，离线识别无有效 IP、同 IP 多 MAC、同 MAC 多 IP、未报备入网、IP-MAC 不符、未纳入 9200、疑似废弃下线等风险。疑似废弃下线会额外检查同 MAC 在 IP 管理平台或 9200 平台是否仍有近期活跃证据，避免把换 IP 的活跃终端误判为废弃。比对判定只使用 IP/MAC 和 9200 状态字段，责任人、部门、安装位置、设备用途和设备类型仅作为风险定位信息展示和导出；同 MAC 多 IP、同 IP 多 MAC 支持聚合证据链展示；每条风险展示判定依据；表头支持 IP、MAC、接入设备和更新时间排序；风险结果支持离线 OUI 厂商识别；每次分析会保存历史记录，风险表格按滚动懒加载显示，筛选和导出仍基于全量结果。
- 首页看板：新增当前网络告警卡片，显示高风险、中风险、采集失败和已采集设备数量。
- 离线绑定导入：预览时区分原始发现绑定数、去重后的唯一终端数和重复记录数，避免把重复备份误认为新增终端。
- 定时任务可调：可在网页用单个时间框设置自动备份、终端更新、自动数据包导出和日志采集时间。
- 审计日志：首页看板和审计记录可追踪手动操作、定时任务、终端更新和设备保存结果。

## 运行方式

推荐使用本机现有 Conda 环境：

```powershell
conda activate sygaSwitchAdmin_base
cd E:\miniconda3\app\sygaSwitchAdmin
python run_server.py
```

默认访问地址：

```text
http://127.0.0.1:8080
```

## 老服务器源码覆盖更新

从新版源码包更新老服务器时，建议只覆盖程序源码，不要清空整个目录后再解压。

推荐步骤：

1. 先停止老服务器上的 NetMaster 程序。
2. 备份老服务器当前源码目录，尤其保留：
   - `net_assets.db`
   - `net_assets.key`
   - `backups/`
   - 其他手工保存的数据、导出文件或历史配置目录
3. 将新版 `sygaSwitchAdmin_source_v版本号_日期.zip` 解压到老服务器源码目录。
4. 选择覆盖同名程序文件。
5. 不要覆盖或删除老服务器已有的 `net_assets.db` 和 `net_assets.key`。
6. 重新启动：
   ```powershell
   conda activate sygaSwitchAdmin_base
   cd D:\miniconda3\app\sygaSwitchAdmin
   python run_server.py
   ```

说明：源码包默认不携带数据库、密钥、日志、历史缓存和设备资产数据。只要保留老服务器原有的 `net_assets.db` 与 `net_assets.key`，设备资产、用户、审计日志和业务数据就不会因覆盖源码而丢失。

默认管理员账号在首次初始化数据库时创建：

```text
admin / admin888
```

上线后应立即在网页中修改密码。

## 数据库说明

数据库文件：

```text
net_assets.db
```

服务启动链路：

```text
run_server.py -> app.py -> database.py -> init_db() + upgrade_db()
```

因此只要启动 `run_server.py`，数据库表和新增字段会自动补齐。

核心表：

- `switches`：交换机资产。
- `mac_bindings`：已绑定终端本地状态表。
- `audit_logs`：审计日志。
- `system_settings`：系统设置，例如定时备份后是否自动保存配置、终端更新超时、并发数、端口保护关键词、SNMP 只读团体名。

## 重要定时任务

- 默认 `02:37`：自动备份交换机配置，可在健康检查页面调整。
- 默认 `03:20`：自动执行终端更新，可在健康检查页面调整。
- 默认 `04:10`：自动导出数据包到 `data_packages/`，可在健康检查页面关闭或调整。
- 默认 `04:40`：自动采集每台交换机日志并生成告警分析报告，可在健康检查页面关闭或调整。

如果“定时备份成功后自动保存设备配置”开启，备份成功后会继续向设备下发保存配置命令。该开关位于“系统设置”页面中。

该工具会直接登录并修改交换机配置。批量部署、终端漫游、保存配置等操作应先在测试端口验证，再用于生产网。

## 运行环境迁移说明

- 推荐 Python 3.11 / Conda 环境。
- 新机器可在项目目录执行 `pip install -r requirements.txt` 安装依赖。
- `requirements.txt` 已整理为可迁移格式，不再包含 `file:///C:/miniconda3/conda-bld/...` 这类仅当前机器存在的本地构建路径。
- Excel 导入/导出依赖 `openpyxl==3.1.5`，已写入依赖清单。
